[로컬 실행]
1. Node.js 설치 후 이 폴더에서: npm install
2. (선택) .env에 VITE_GOOGLE_SHEET_WEBHOOK_URL=본인 웹앱 /exec 주소
3. npm run dev → 브라우저 주소로 접속

[제출 JSON에 자동 포함되는 항목(학생 화면 비표시)]
- clientDeviceInfo: 한 줄 문자열. 예) "노트북·PC(추정) · Windows 10/11", "모바일(추정) · Android"
※ 접속 형태·OS는 UA 등으로 추정하므로 100% 정확하지 않습니다. 시트 마지막 열(기기 또는 OS)에 data.clientDeviceInfo 를 appendRow 하면 됩니다.

[제출 동작]
- 학생이 제출하면 JSON이 구글 앱스 스크립트 웹앱(POST)으로만 전송됩니다. Firebase는 사용하지 않습니다.
- 앱스 스크립트 전체 예시: 이 폴더의 google-apps-script/Code.gs 를 스프레드시트「확장 프로그램 → Apps Script」에 붙여넣고 저장·배포하세요.

[시트 구분]
- 3학년: payload.sheetName = "시트1"
- 2학년: payload.sheetName = "시트2"
- 앱스 스크립트는 getSheetByName(data.sheetName) 후 학년별로 appendRow 열 구성이 다릅니다.

[배포]
- 정적 호스팅만 필요합니다: npm run build → dist 폴더를 Netlify, GitHub Pages, 학교 서버 등에 올리면 됩니다.
- 빌드 시 웹앱 URL을 넣으려면 배포 전 .env에 VITE_GOOGLE_SHEET_WEBHOOK_URL 설정 후 build 하세요.

[GitHub]
- .env는 Git에 올리지 마세요 (.gitignore)

[참고]
- 소스: src/App.jsx
- 루트 수행평가제출.html은 안내용이며, 실제 앱은 npm run dev / build 결과입니다.
- 데이터셋: /datasets(허브) → /datasets/assessment(수행평가 제출용), /datasets/practice(연습용). npm run dev / build 시 sync로 dataset/practice|assessment/grade2·grade3가 public/dataset/에 복사됩니다.
